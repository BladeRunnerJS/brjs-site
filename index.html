Divide & Conquer your Web Apps

BladeRunner is a development toolkit and lightweight framework for building web applications consisting of one or more components called Blades. It comes complete with some seriously useful tools which make it easy to develop, test and deploy your app.

Components

An application built with BladeRunner consists of one or more Blades. Each Blade can be made up of HTML, JavaScript, CSS and other assets.

Blades can be run in isolation during development making them easy to develop and test. Inter-blade communication via an event-hub.

Developer Tools

Workbenches allow Blades to be run and debugged independently of the app.

Workbench widgets allow you to interact with the stub services you use in the workbench. For example you can view your component in different states by sending your component mock events.

Clean Code

Code is organized cleanly using classes, with one class per file, and JavaScript interface support is provided to help with creating large scalable code-bases.

Services are a first-class concept that allow your application to be written before the back-end is finished, and can be replaced by mocks in your tests.
