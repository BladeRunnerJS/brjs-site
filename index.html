Divide & conquer your web apps

BladeRunnerJS (BRJS) is a development toolkit and lightweight framework for building modular large-scale HTML5 single page web apps. It consists of a set of conventions plus associated tools and libraries that make it easy to develop, test, deploy and maintain large JavaScript apps.

{% include docs/concepts/blade_overview.html %}

Workbenches

Workbenches allow Blades to be developed, debugged and run independently of the app.

Workbench Tools allow you to interact with the Blade through stubbed services and via the EventHub. For example you can view your Blade in different states by sending it mock events via a mock events Workbench Tool.

Clean Code

Code is organized cleanly using classes, with one class per file, and JavaScript interface support is provided to help with creating large scalable codebases.

Services are a first-class concept that allow your application to be written before the back-end is finished, and can be replaced by mocks in your tests.

Scaffolding

Build out the structure of your app quickly using the BRJS scaffolding feature. Create Apps, Blades, BladeSets, Aspects, Classes, Interfaces, Tests and more via the CLI or web dashboard.

Dependency Analysis

The BRJS bundling system analyzes your code and ensures that only the JavaScript files that are used within your application are bundled during the build and deployment steps.

Scale to Multiple Teams

One of the main benefits of using BRJS is that is ensures your application will scale to multiple teams. By seperating functionality into Blades and libraries, and by using the EventHub and Services, it massively reduces the likelihood of feature incompatibility and code conflicts.

Efficient UI Testing

When developing complex UIs you can end up with several hundred detailed behavioural tests. By using our MVVM presentation library you can test the UI via the "view model", without having to touch the DOM. Hundreds of UI tests can therefore be executed in just a few minutes.

Web Dashboard

Not everybody wants to spend all their time on the command line. So, BRJS ships with a web dashboard which offers most of the functionality that the CLI offers via a simple web user interface.

Plugins

BRJS gives you everything you need out of the box. However, it's great to be able to extend and customise functionality to meet different needs. Plugins provide a framework to extend the BRJS toolkit with your own commands, file transforms, bundling and minimization strategies and much more.
